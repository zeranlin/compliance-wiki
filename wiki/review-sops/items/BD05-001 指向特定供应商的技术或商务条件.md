# BD05-001 指向特定供应商的技术或商务条件

## 标准检查点
指向特定供应商的技术或商务条件

## 检查点定义
本检查点用于审查采购文件是否通过技术要求、商务条件、服务体系、接口协议、兼容对象、原厂体系、合作关系、特定平台或特定服务标准，实际指向某一供应商、供应商关联体系或少数特定主体。

## 权威依据
实施条例、采购需求管理办法、38号文、四类专项整治

## 适用边界
适用于通过技术、商务、服务、接口、平台、原厂体系或合作关系变相指向特定供应商的审查。

### 二、组合触发规则

#### 强触发
- 表头词 + 指向词 + 后果词。
- 技术或商务条件要求对接、兼容、接入、纳入特定供应商系统或平台。
- 要求由特定供应商、特定厂家、特定原厂体系出具证明、承诺或服务。
- 要求投标人具有特定厂商合作伙伴、授权服务商、生态成员等身份。
- 条款未直接写品牌，但通过接口、协议、服务体系、平台账号或合作关系限定供应商。

#### 中触发
- 出现本检查点对象词，并与资格、评分、响应、证明材料、合同履约、验收付款或监管判断存在同段或相邻条款关系，但后果、必要性或替代路径不清。
- 同一事项在公告、采购文件、评分办法、合同、附件或响应格式中口径不一致，需要进一步判断条款优先级和实际影响。
- 条款可能对应项目特殊需要或法定监管要求，但文件没有写清依据、范围、比例、期限、标准或证明路径。

#### 弱触发
- 仅说明现有系统环境、已有设备或对接背景。
- 要求满足开放接口、通用协议或行业标准。
- 中标后根据采购人现有系统进行合理对接，不影响投标阶段。

#### 不触发
- 采购文件只是描述采购人现状，不把现状作为投标门槛。
- 要求兼容通用标准、公开协议或依法必须对接的政务平台。
- 允许等效替代、开放接口或多品牌多供应商满足。

### 三、审查推理规则

#### 命中规则
以下 A、B、C 同时满足，即进入命中候选：
- A：存在特定供应商、平台、接口、原厂体系、合作关系或专属资源要求。
- B：该要求影响资格、符合性、评分、响应有效性或材料认可。
- C：文件未说明充分必要性，或未提供等效替代、开放接口、中标后适配等非排他路径。
通常满足以下任意 3 项及以上，认定命中本检查点：
- 条款实际锁定特定供应商、厂家体系、平台或服务生态。
- 不满足即投标无效、负偏离、不得分或不予认可。
- 要求发生在投标阶段，而非中标后适配阶段。
- 文件缺少开放接口、等效替代或第三方适配路径。
- 条款评价的是供应商与特定主体的关系，而非实际履约能力。
- 相关要求不是法律法规、强制标准或安全合规所必需。
- 指定特定公司、平台、系统、接口、协议或服务体系名称。
- 要求特定供应商出具授权、证明、承诺或接口开放文件。
- 与指定品牌、原厂授权、唯一参数组合同时出现。

#### 待人工复核规则
- 候选条款已经形成实质疑点，但仍需补充项目特殊必要性、法律依据、审批文件、附件、合同、平台日志、投标文件、交易数据或人工事实核验。
- 条款对资格、评分、响应有效性或履约结果的影响不清，或者多个章节存在冲突但无法仅凭当前文本确定优先级。
- 条款可能属于法定监管、强制标准、安全合规、应急、涉密等特殊场景，但文件未写明充分依据或替代路径。

#### 不命中规则
任一满足，应排除或降级为待人工复核：
- 仅描述采购人现有环境，不作为投标条件。
- 要求兼容公开标准、通用协议或法定政务平台，且不限定供应商。
- 明确允许等效替代、开放接口、第三方适配或中标后完成对接。
- 涉及国家安全、涉密、数据安全、系统连续性等特殊必要场景，且文件说明理由。
- 单一来源或法定特殊采购方式下已有合法论证。

#### 专项判断尺子
- 先判断候选条款是否落入本检查点对象范围。
- 再判断该事项是否影响资格、评分、响应有效性、政策适用、合同履约、验收付款或监管判断。
- 最后检查文件是否提供合法、必要、清楚、可核验的理由或替代路径。

### 四、质量控制与验证规则

#### 必须提取的证据
对每个候选条款，必须读取：
- 当前条款全文。
- 当前条款前 5 行和后 10 行。
- 如果是表格，读取参数名称、是否星号、偏离后果、证明材料列和备注列。
- 采购人现有系统、设备、平台或历史供应商背景。
- 是否允许等效替代、开放协议、第三方接口或中标后适配。
- 条款是否影响资格、符合性、评分或投标有效性。

#### 必须检查的排除条件
- 仅描述采购人现有环境，不作为投标条件。
- 要求兼容公开标准、通用协议或法定政务平台，且不限定供应商。
- 明确允许等效替代、开放接口、第三方适配或中标后完成对接。
- 涉及国家安全、涉密、数据安全、系统连续性等特殊必要场景，且文件说明理由。
- 单一来源或法定特殊采购方式下已有合法论证。
- “项目需与采购人现有业务系统对接，中标后由供应商按开放接口规范完成适配。”。
- “产品应支持通用 API 或国家标准协议。”。
- “采购人现有设备为某品牌，仅作为现场环境说明，不作为投标限制。”。

#### 常见漏判
- 只检索显性关键词，没有把同义近义词、限制动作词和后果词组合起来判断。
- 候选词出现在表格证明材料列、备注列或响应格式中，但没有回读对应评分项、资格项或符合性审查项。
- 只看到通用合法表述，忽略专用条款、评分表、附件或响应格式中的相反约束。

#### 常见误判
- 只看到关键词就命中，没有确认资格、评分、响应有效性、履约验收、付款或监管后果。
- 把项目地点、履约地点、背景说明、模板条款或法定通用要求误判为限制条件。
- 未检查排除条件、替代路径、法定依据或项目特殊必要性。

#### 最小验证样例
正例：
- “投标产品须接入某公司平台并提供该公司出具的接口证明，否则视为无效响应。”
- “供应商须为某厂商授权服务商，未提供证明不得参与投标。”
- “系统必须使用某供应商专属协议，且不接受其他协议适配。”

反例：
- “项目需与采购人现有业务系统对接，中标后由供应商按开放接口规范完成适配。”
- “产品应支持通用 API 或国家标准协议。”
- “采购人现有设备为某品牌，仅作为现场环境说明，不作为投标限制。”

#### 执行步骤
1. 定位技术要求、商务要求、系统对接、兼容性、原厂服务和证明材料条款。
2. 用特定供应商、原厂、平台、接口、合作伙伴、授权服务商等词召回候选条款。
3. 读取上下文，判断该表述是背景说明还是投标条件。
4. 判断是否影响资格、符合性、评分或响应有效性。
5. 使用指向性判断尺子回答五问。
6. 先执行排除条件，排除开放标准、合理兼容和中标后适配。
7. 再判断基础命中条件、核心命中条件和强化命中情形。
8. 最后输出命中、待人工复核或不命中。
