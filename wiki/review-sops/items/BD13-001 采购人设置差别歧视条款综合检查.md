# BD13-001 采购人设置差别歧视条款综合检查

## 标准检查点
采购人设置差别歧视条款综合检查

## 检查点定义
综合审查专项整治关注的所有制、地域、规模、业绩、证书、荣誉、平台库等差别歧视条款。

本检查点是“专项总控/综合佐证型检查点”，不是替代 BD01-BD09 的独立定性工具。能够归入地域、规模、业绩、证书、荣誉、评分相关性、证明材料等专门检查点的，应优先由对应专门检查点定性；本检查点只负责把多个专门风险线索归纳为差别歧视方向。

## 权威依据
政府采购法及实施条例、87号令、采购需求管理办法、促进政府采购公平竞争优化营商环境通知、政府采购促进中小企业发展管理办法、政府采购质疑和投诉办法、四类专项整治监管口径。

## 适用边界
专项综合检查用于兜底发现差别歧视条款；若能归入 BD01-BD09 专项检查点，应优先细分。

## 审查执行说明书（3+1）

### 一、定位词体系

#### 显性关键词
- 差别待遇。
- 歧视待遇。
- 本地。
- 特定区域。
- 注册资本。
- 业绩。
- 荣誉。
- 证书。
- 平台。
- 备选库。

#### 同义近义词
- 采购公告、采购文件、投标人须知、评分办法、合同草案、附件和响应格式。
- 与本检查点标题直接相关的资格、评分、政策、程序、合同、验收、付款或监管线索条款。
- 平台。
- 备选库。

#### 限制动作词
- 必须。
- 须。
- 应当。
- 不得。
- 不接受。
- 不予认可。
- 提供。
- 提交。
- 满足。
- 未提供。
- 未满足。

#### 后果词
- 资格条件。
- 符合性审查。
- 实质性响应。
- 得分。
- 加分。
- 价格扣除。
- 预留份额。
- 无效投标。
- 履约。
- 验收。
- 付款。
- 监管线索。

#### 排除词
- 仅为背景说明、法定通用表述、已有明确补充条款、或不影响供应商参与和履约结果时，不命中。
- 条款已有明确合法依据，且与采购标的、项目特点和履约风险相适应。
- 其他章节已经补足适用对象、比例、期限、程序、标准、金额、范围或数据解释。
- 需要外部审批、预算批复、交易日志、投标文件或人工事实核验才能确认时，输出待人工复核。
- 仅为证书、职称、学历、项目经验与采购需求相关性不足，应优先交由 BD06、BD08、BD09 判断；本检查点不单独命中。
- 专门检查点为待人工复核的同一证据，本检查点不得升级为命中。
- 候选理由只能写“需复核必要性、需结合项目特点判断、可能排他”，但没有明确歧视类型和排他后果时，不命中或待人工复核。
- 文件只是提示依法执行相关政策或程序，未形成额外限制或错误口径。
- 条款已经明确对象、范围、比例、期限、程序、标准和证明材料，且与项目事实相适应。
- 数据型检查点只有单一弱信号或缺少交易数据，不能直接定性。
- 要求项目负责人具备 PMP、职称、学历或同类经验，但是否必要需要结合项目复杂度判断，且尚不能明确归入地域、规模、特定机构或特定客户限制。
- 要求通用体系认证、检测报告或合同业绩，但主要问题是与采购需求相关性不足或证明材料负担，尚不能明确构成差别歧视，应交由 BD06 或 BD09 判断。

### 二、组合触发规则

#### 强触发
- 出现对象词，并且同一条款或相邻条款出现资格、评分、响应、价格评审、合同履约、验收付款或监管后果。
- 条款直接使用“必须、须、不得、无效、不得分、不予认可、按采购人要求、以实际为准”等约束性表达。
- 条款把本检查点对象作为参与采购、获得优惠、取得分数、通过审查、签订合同、获得付款或形成监管判断的条件。
- 同一事项在公告、采购文件、合同、附件或响应格式中口径不一致。
- 差别待遇 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 歧视待遇 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 本地 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 特定区域 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 注册资本 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。

#### 中触发
- 出现本检查点对象词，并与资格、评分、响应、证明材料、合同履约、验收付款或监管判断存在同段或相邻条款关系，但后果、必要性或替代路径不清。
- 同一事项在公告、采购文件、评分办法、合同、附件或响应格式中口径不一致，需要进一步判断条款优先级和实际影响。
- 条款可能对应项目特殊需要或法定监管要求，但文件没有写清依据、范围、比例、期限、标准或证明路径。

#### 弱触发
- 仅出现对象词，但没有资格、评分、响应有效性、政策、履约、验收付款或监管后果时，只作背景记录。

#### 不触发
- 关键词只出现在法规提示、背景介绍或非约束性说明中。
- 文件已给出明确合法口径，且没有与其他章节冲突。
- 该事项明显属于其他更专门检查点，应转入对应 BD 检查点。
- 仅是证书、学历、职称、项目管理经验等与履约相关性存在疑问，但尚不能直接归入明确差别歧视类型的条款。
- 仅需结合项目特点判断必要性的条款，不作为本检查点命中候选。

### 三、审查推理规则

#### 命中规则
以下 A、B、C 同时满足，即进入命中候选：
- A：候选条款能够明确映射到一种具体差别歧视类型，例如地域、所有制、规模、特定客户类型、特定机构证书、特定协会荣誉、平台库准入。
- B：该条款直接影响资格、符合性审查、评分、响应有效性、价格评审或供应商参与机会。
- C：文件未说明该限制与采购标的、履约风险、法律法规或政策要求之间存在充分必要关系，且未提供同等替代路径。
通常应同时满足以下条件，才认定命中本检查点：
- 候选证据与检查点标题直接相关。
- 已明确映射到具体差别歧视类型，而不是泛泛认为“不合理”。
- 风险后果不是抽象可能，而是会影响供应商参与、评审、政策适用、救济、履约验收、付款或监管判断。
- 文件内部或交易数据内部无法自行消除疑点。
- 不存在明确法定依据、项目特殊必要性或充分替代路径。
- 若同一证据只被本检查点命中、没有专门检查点命中支撑，应谨慎降为待人工复核。

#### 待人工复核规则
- 候选条款已经形成实质疑点，但仍需补充项目特殊必要性、法律依据、审批文件、附件、合同、平台日志、投标文件、交易数据或人工事实核验。
- 条款对资格、评分、响应有效性或履约结果的影响不清，或者多个章节存在冲突但无法仅凭当前文本确定优先级。
- 条款可能属于法定监管、强制标准、安全合规、应急、涉密等特殊场景，但文件未写明充分依据或替代路径。

#### 不命中规则
任一满足，应排除或降级：
- 仅为背景说明、法定通用表述、已有明确补充条款、或不影响供应商参与和履约结果时，不命中。
- 条款已有明确合法依据，且与采购标的、项目特点和履约风险相适应。
- 其他章节已经补足适用对象、比例、期限、程序、标准、金额、范围或数据解释。
- 需要外部审批、预算批复、交易日志、投标文件或人工事实核验才能确认时，输出待人工复核。
- 仅为证书、职称、学历、项目经验与采购需求相关性不足，应优先交由 BD06、BD08、BD09 判断；本检查点不单独命中。
- 专门检查点为待人工复核的同一证据，本检查点不得升级为命中。
- 候选理由只能写“需复核必要性、需结合项目特点判断、可能排他”，但没有明确歧视类型和排他后果时，不命中或待人工复核。

#### 专项判断尺子
按以下五问执行：

1. 候选条款或数据是否确实落入“采购人设置差别歧视条款综合检查”的对象范围？
2. 该事项是否影响供应商参与、价格评审、政策优惠、响应有效性、评分、履约、验收、付款或监管判断？
3. 文件或数据是否提供了清楚、合法、可核验的适用口径？
4. 是否存在与项目类型、采购标的、合同履约或监管数据不匹配的地方？
5. 是否有明确排除理由，例如法定要求、项目特殊需要、其他章节已补足或证据不足？

判断规则：
- 第 1、2 问为“是”，第 3 或第 4 问显示口径错误、不清或不匹配，且无排除理由时，进入命中候选。
- 证据不足但存在实质疑点时，输出待人工复核。
- 仅有关键词、没有实质后果时，不命中。
- 仅有“需结合项目特点判断必要性”的候选，不能命中，只能待人工复核或不命中。
- 如果不能明确映射到具体差别歧视类型，不能命中。

### 四、质量控制与验证规则

#### 必须提取的证据
对每个候选条款，必须读取：
- 当前条款全文及前后 10 行。
- 所在章节标题、表头、分值列、证明材料列、备注列。
- 与同一事项相关的公告、须知、评分办法、合同、附件和响应格式。
- 如为数据型检查点，必须读取至少两个供应商或两组交易数据之间的可比字段。

#### 必须检查的排除条件
- 仅为背景说明、法定通用表述、已有明确补充条款、或不影响供应商参与和履约结果时，不命中。
- 条款已有明确合法依据，且与采购标的、项目特点和履约风险相适应。
- 其他章节已经补足适用对象、比例、期限、程序、标准、金额、范围或数据解释。
- 需要外部审批、预算批复、交易日志、投标文件或人工事实核验才能确认时，输出待人工复核。
- 仅为证书、职称、学历、项目经验与采购需求相关性不足，应优先交由 BD06、BD08、BD09 判断；本检查点不单独命中。
- 专门检查点为待人工复核的同一证据，本检查点不得升级为命中。
- 候选理由只能写“需复核必要性、需结合项目特点判断、可能排他”，但没有明确歧视类型和排他后果时，不命中或待人工复核。
- 文件只是提示依法执行相关政策或程序，未形成额外限制或错误口径。
- 条款已经明确对象、范围、比例、期限、程序、标准和证明材料，且与项目事实相适应。
- 数据型检查点只有单一弱信号或缺少交易数据，不能直接定性。

#### 常见漏判
- 只检索显性关键词，没有把同义近义词、限制动作词和后果词组合起来判断。
- 候选词出现在表格证明材料列、备注列或响应格式中，但没有回读对应评分项、资格项或符合性审查项。
- 只看到通用合法表述，忽略专用条款、评分表、附件或响应格式中的相反约束。

#### 常见误判
- 只看到关键词就命中，没有确认资格、评分、响应有效性、履约验收、付款或监管后果。
- 把项目地点、履约地点、背景说明、模板条款或法定通用要求误判为限制条件。
- 未检查排除条件、替代路径、法定依据或项目特殊必要性。

#### 最小验证样例
正例：
- 要求供应商必须在本地注册、本地设分支机构、本地纳税，作为资格、评分或无效投标条件。
- 要求提供特定行政区域、特定客户类型或特定行业的业绩，其他同类业绩不得分，且无必要性说明。
- 要求认证证书必须由某一特定认证机构出具，排除其他合法认证机构或同等证明。
- 要求进入某平台库、名录库、备选库后才能投标、得分或通过审查。
- 要求特定协会荣誉、奖项、排名作为高分评分因素，且不接受同等能力证明。

反例：
- 文件只是提示依法执行相关政策或程序，未形成额外限制或错误口径。
- 条款已经明确对象、范围、比例、期限、程序、标准和证明材料，且与项目事实相适应。
- 数据型检查点只有单一弱信号或缺少交易数据，不能直接定性。
- 要求项目负责人具备 PMP、职称、学历或同类经验，但是否必要需要结合项目复杂度判断，且尚不能明确归入地域、规模、特定机构或特定客户限制。
- 要求通用体系认证、检测报告或合同业绩，但主要问题是与采购需求相关性不足或证明材料负担，尚不能明确构成差别歧视，应交由 BD06 或 BD09 判断。

#### 执行步骤
1. 先检索对象词、后果词和组合信号。
2. 提取候选条款或交易数据，并读取上下文。
3. 先判断是否能归入更专门检查点；能归入时，本检查点只做综合佐证，不扩大定性。
4. 判断是否存在明确差别歧视类型，并写出类型名称。
5. 判断是否存在排他性表达和资格、评分、无效投标等后果。
6. 执行专项判断尺子五问。
7. 检查排除条件和其他章节是否补足。
8. 若只是“需结合项目特点判断必要性”，不得命中。
9. 按命中、待人工复核、不命中输出结论。
