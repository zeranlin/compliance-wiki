所属问题域：[[checkpoint-domains/BD11 程序、信息公开与救济|BD11 程序、信息公开与救济]]

问题域说明：本检查点属于程序、信息公开与救济问题域，用于识别“公告与采购文件关键口径不一致”相关风险，并把判断过程压缩成可由小模型执行的 SOP。

## 标准检查点
公告与采购文件关键口径不一致

## 检查点定义
审查公告、投标邀请、采购文件正文、供应商须知在预算、资格、采购方式、评标方法、时间地点等关键口径是否不一致。

## 审查问题句
是否存在“公告与采购文件关键口径不一致”风险，并对政府采购公平竞争、政策适用、程序透明、履约验收或监管判断产生实质影响？

## 对应 A 边界
- [[boundaries/A-BD-11|A-BD-11]]：本检查点应回到对应 A 边界判断，不得脱离采购类型、项目事实和证据直接定性。

## 权威依据
政府采购法及实施条例、87号令、采购需求管理办法、促进政府采购公平竞争优化营商环境通知、政府采购促进中小企业发展管理办法、政府采购质疑和投诉办法、四类专项整治监管口径。

## 审查对象
- 采购公告、采购文件、投标人须知、评分办法、合同草案、附件和响应格式。
- 与本检查点标题直接相关的资格、评分、政策、程序、合同、验收、付款或监管线索条款。

## 查找范围
- 采购公告。
- 投标邀请。
- 投标人须知。
- 资格条件。
- 符合性审查。
- 评分办法。
- 政策功能章节。
- 采购需求。
- 合同草案。
- 附件和响应格式。

## 定位关键词

### 对象词
- 公告。
- 采购文件。
- 投标邀请。
- 预算。
- 最高限价。
- 资格条件。
- 评标方法。
- 开标时间。
- 提交截止。
- 采购方式。

### 后果词
- 资格条件。
- 符合性审查。
- 实质性响应。
- 得分。
- 加分。
- 价格扣除。
- 预留份额。
- 无效投标。
- 履约。
- 验收。
- 付款。
- 监管线索。

### 组合信号
- 公告 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 采购文件 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 投标邀请 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 预算 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 最高限价 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。

## 候选召回规则
### 强召回
满足以下任一情形，应召回为候选条款：
- 出现对象词，并且同一条款或相邻条款出现资格、评分、响应、价格评审、合同履约、验收付款或监管后果。
- 条款直接使用“必须、须、不得、无效、不得分、不予认可、按采购人要求、以实际为准”等约束性表达。
- 条款把本检查点对象作为参与采购、获得优惠、取得分数、通过审查、签订合同、获得付款或形成监管判断的条件。
- 同一事项在公告、采购文件、合同、附件或响应格式中口径不一致。

### 弱召回
仅出现对象词，但没有约束后果、评分后果、政策后果、履约后果或监管数据支撑时，只作背景记录。

### 不召回
以下情形不进入本检查点候选：
- 关键词只出现在法规提示、背景介绍或非约束性说明中。
- 文件已给出明确合法口径，且没有与其他章节冲突。
- 该事项明显属于其他更专门检查点，应转入对应 BD 检查点。

## 上下文读取规则
对每个候选条款，必须读取：
- 当前条款全文及前后 10 行。
- 所在章节标题、表头、分值列、证明材料列、备注列。
- 与同一事项相关的公告、须知、评分办法、合同、附件和响应格式。
- 如为数据型检查点，必须读取至少两个供应商或两组交易数据之间的可比字段。

## 条款属性判断
判断候选条款属于以下哪类属性：
- 资格条件。
- 符合性审查。
- 评分因素。
- 政策适用。
- 信息公开。
- 质疑投诉。
- 合同履约。
- 验收付款。
- 交易成本。
- 监管线索。
- 模板残留。

## 专项判断尺子
按以下五问执行：

1. 候选条款或数据是否确实落入“公告与采购文件关键口径不一致”的对象范围？
2. 该事项是否影响供应商参与、价格评审、政策优惠、响应有效性、评分、履约、验收、付款或监管判断？
3. 文件或数据是否提供了清楚、合法、可核验的适用口径？
4. 是否存在与项目类型、采购标的、合同履约或监管数据不匹配的地方？
5. 是否有明确排除理由，例如法定要求、项目特殊需要、其他章节已补足或证据不足？

判断规则：
- 第 1、2 问为“是”，第 3 或第 4 问显示口径错误、不清或不匹配，且无排除理由时，进入命中候选。
- 证据不足但存在实质疑点时，输出待人工复核。
- 仅有关键词、没有实质后果时，不命中。

## 专用 SOP 边界
本检查点必须建立在“至少两个可比来源”之上，不能只凭采购文件中引用公告、引用投标邀请、引用供应商须知，就认定公告与采购文件不一致。

可比来源包括：
- 采购公告或更正公告。
- 投标邀请。
- 采购文件项目专用条款。
- 投标人须知前附表。
- 资格条件章节。
- 评分办法或评标方法章节。
- 采购需求摘要。
- 响应截止、开标、解密、演示、踏勘等程序安排。

必须比对的关键口径：
- 项目名称、项目编号、采购包号。
- 预算金额、最高限价、报价上限。
- 采购方式、评标方法、价格分算法。
- 供应商资格条件、联合体、分包、中小企业政策。
- 获取文件、提交截止、开标、解密、澄清修改、现场演示或踏勘时间地点。
- 采购需求范围、服务期限、交付地点、合同履行期限。

判断口径：
- 若只有采购文件，未提供公告或更正公告原文，且文件只是写“资格要求详见公告”“具体时间详见公告”等引用关系，候选条款不作为业务风险输出；可在执行记录中说明“缺少公告原文，无法比对”，但候选结论应为不命中或不进入业务报告，不得命中。
- 若两个来源对同一关键事项写法不同，但文件明确规定以更正公告、项目专用条款或后发布文件为准，且无供应商误导风险，通常不命中。
- 若两个来源对资格、预算、最高限价、截止时间、评标方法等关键事项存在实质冲突，且未说明优先级或更正关系，应命中。
- 若差异只是简称、格式、章节引用、法条名称、非关键说明，不命中。
- 若候选证据来自通用模板、响应格式、法定提示，不作为不一致证据；必须找到项目专用信息之间的冲突。

证据要求：
- 命中必须成对引用两个来源，写明“来源 A 怎么写、来源 B 怎么写、冲突影响什么”。
- 待人工复核必须写明缺少哪个来源，例如“缺采购公告原文”“缺更正公告”“缺投标邀请专用页”。
- 不命中必须说明两个来源口径一致，或只是引用关系、模板说明、非关键差异。

## 基础命中条件
以下 A、B、C 同时满足，即进入命中候选：
- A：存在与“公告与采购文件关键口径不一致”直接相关的条款、要求或交易数据线索。
- B：条款直接影响资格、评分、响应有效性、价格评审、合同履约、验收付款、供应商权利或政府采购政策适用。
- C：文件或数据未提供清楚、合法、可核验的解释，或存在明显错配、冲突、过度限制、不合理负担或监管异常。

## 命中条件
通常应同时满足以下条件，才认定命中本检查点：
- 候选证据与检查点标题直接相关。
- 风险后果不是抽象可能，而是会影响供应商参与、评审、政策适用、救济、履约验收、付款或监管判断。
- 文件内部或交易数据内部无法自行消除疑点。
- 不存在明确法定依据、项目特殊必要性或充分替代路径。

## 排除条件
任一满足，应排除或降级：
- 仅为背景说明、法定通用表述、已有明确补充条款、或不影响供应商参与和履约结果时，不命中。
- 条款已有明确合法依据，且与采购标的、项目特点和履约风险相适应。
- 其他章节已经补足适用对象、比例、期限、程序、标准、金额、范围或数据解释。
- 需要外部审批、预算批复、交易日志、投标文件或人工事实核验才能确认时，输出待人工复核。
- 缺少公告、更正公告或投标邀请原文，无法形成两个来源比对时，不得命中，只能待人工复核或不命中。
- 采购文件只是引用公告作为资格条件、时间节点或程序安排依据，且未出现另一个相冲突口径时，不命中，不进入业务风险问题。
- 差异已有更正公告、补遗文件、项目专用条款优先级说明消除时，不命中。

## 判断结果分流
- 命中：基础命中条件 A、B、C 均满足，且无排除条件。
- 待人工复核：存在实质疑点，但需要外部政策、附件、审批、交易数据或人工事实核验。
- 不命中：只有关键词或背景表述，或者合法口径已经明确。

## 正例
- 条款将本检查点对象直接设置为资格、评分、价格优惠、响应有效性、付款或监管判断条件，但适用对象、比例、期限、标准、数据来源或责任边界不清。
- 同一事项在公告、正文、合同、附件或响应格式中口径不同，供应商无法据此稳定响应。
- 数据型检查点中，多个供应商之间出现同账户、同设备、同作者、规律性报价等异常一致，并有两个以上字段交叉印证。
- 公告写“允许联合体投标”，采购文件资格条件写“不接受联合体投标”，且无更正公告说明。
- 公告写提交截止时间为某日 9:00，采购文件投标人须知写某日 14:30，且未说明哪个为准。
- 公告写最高限价 100 万元，采购文件报价上限或预算表写 120 万元，且无更正关系。

## 反例
- 文件只是提示依法执行相关政策或程序，未形成额外限制或错误口径。
- 条款已经明确对象、范围、比例、期限、程序、标准和证明材料，且与项目事实相适应。
- 数据型检查点只有单一弱信号或缺少交易数据，不能直接定性。
- 采购文件写“资格要求详见公告”，但当前待审文件未提供公告原文且没有第二个冲突来源，属于缺少比对对象，不作为业务风险输出。
- 公告与采购文件用语略有不同，但项目名称、编号、预算、资格、时间、评标方法等关键事项一致。
- 更正公告已经明确替换原公告或采购文件中的相关条款。

## 审查步骤
1. 先检索对象词、后果词和组合信号。
2. 提取候选条款或交易数据，并读取上下文。
3. 判断条款属性和影响环节。
4. 执行专项判断尺子五问。
5. 检查排除条件和其他章节是否补足。
6. 按命中、待人工复核、不命中输出结论。
7. 对每个疑点必须建立“来源 A vs 来源 B”比对表；没有两个来源不得命中。
8. 先查是否存在更正公告、补遗、澄清或优先级条款；能消除冲突的，不命中。
9. 如果只是“详见公告”但没有公告原文，不要把“无法比对”包装成业务风险；应判为不命中或仅在执行记录中提示资料缺口。

## 结论表达
- 命中时写明：原文或数据证据、错误口径、影响环节和法规政策边界。
- 待人工复核时写明：还需要补查的审批、附件、合同、平台日志、投标文件、交易数据或人工事实。
- 不命中时写明：为什么只是关键词出现，或为什么文件口径已经合法明确。

## 适用边界
公告与采购文件关键口径不一致会影响供应商知情和响应；同一事项多处表述不同且无优先级时，应识别。

## 备注
各处预算、资格、时间、地点、方式、评标方法一致；如有差异，文件明确以更正公告或特定章节为准。
