所属问题域：[[checkpoint-domains/BD11 程序、信息公开与救济|BD11 程序、信息公开与救济]]

问题域说明：本检查点属于程序、信息公开与救济问题域，用于识别“质疑提交方式或材料要求过严”相关风险，并把判断过程压缩成可由小模型执行的 SOP。

## 标准检查点
质疑提交方式或材料要求过严

## 检查点定义
审查是否对质疑提交设置不必要的原件、公证、现场、固定格式、授权、盖章或证明材料要求。

## 审查问题句
是否存在“质疑提交方式或材料要求过严”风险，并对政府采购公平竞争、政策适用、程序透明、履约验收或监管判断产生实质影响？

## 对应 A 边界
- [[boundaries/A-BD-12|A-BD-12]]：本检查点应回到对应 A 边界判断，不得脱离采购类型、项目事实和证据直接定性。

## 权威依据
政府采购法及实施条例、87号令、采购需求管理办法、促进政府采购公平竞争优化营商环境通知、政府采购促进中小企业发展管理办法、政府采购质疑和投诉办法、四类专项整治监管口径。

## 审查对象
- 采购公告、采购文件、投标人须知、评分办法、合同草案、附件和响应格式。
- 与本检查点标题直接相关的资格、评分、政策、程序、合同、验收、付款或监管线索条款。

## 查找范围
- 采购公告。
- 投标邀请。
- 投标人须知。
- 资格条件。
- 符合性审查。
- 评分办法。
- 政策功能章节。
- 采购需求。
- 合同草案。
- 附件和响应格式。

## 定位关键词

### 对象词
- 质疑函。
- 原件。
- 现场提交。
- 公证。
- 法定代表人到场。
- 授权委托。
- 固定格式。
- 不予受理。
- 材料不全。

### 后果词
- 资格条件。
- 符合性审查。
- 实质性响应。
- 得分。
- 加分。
- 价格扣除。
- 预留份额。
- 无效投标。
- 履约。
- 验收。
- 付款。
- 监管线索。

### 组合信号
- 质疑函 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 原件 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 现场提交 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 公证 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。
- 法定代表人到场 + 资格 / 评分 / 响应 / 履约 / 验收 / 付款 / 监管后果。

## 候选召回规则
### 强召回
满足以下任一情形，应召回为候选条款：
- 出现对象词，并且同一条款或相邻条款出现资格、评分、响应、价格评审、合同履约、验收付款或监管后果。
- 条款直接使用“必须、须、不得、无效、不得分、不予认可、按采购人要求、以实际为准”等约束性表达。
- 条款把本检查点对象作为参与采购、获得优惠、取得分数、通过审查、签订合同、获得付款或形成监管判断的条件。
- 同一事项在公告、采购文件、合同、附件或响应格式中口径不一致。

### 弱召回
仅出现对象词，但没有约束后果、评分后果、政策后果、履约后果或监管数据支撑时，只作背景记录。

### 不召回
以下情形不进入本检查点候选：
- 关键词只出现在法规提示、背景介绍或非约束性说明中。
- 文件已给出明确合法口径，且没有与其他章节冲突。
- 该事项明显属于其他更专门检查点，应转入对应 BD 检查点。

## 上下文读取规则
对每个候选条款，必须读取：
- 当前条款全文及前后 10 行。
- 所在章节标题、表头、分值列、证明材料列、备注列。
- 与同一事项相关的公告、须知、评分办法、合同、附件和响应格式。
- 如为数据型检查点，必须读取至少两个供应商或两组交易数据之间的可比字段。

## 条款属性判断
判断候选条款属于以下哪类属性：
- 资格条件。
- 符合性审查。
- 评分因素。
- 政策适用。
- 信息公开。
- 质疑投诉。
- 合同履约。
- 验收付款。
- 交易成本。
- 监管线索。
- 模板残留。

## 专项判断尺子
按以下五问执行：

1. 候选条款或数据是否确实落入“质疑提交方式或材料要求过严”的对象范围？
2. 该事项是否影响供应商参与、价格评审、政策优惠、响应有效性、评分、履约、验收、付款或监管判断？
3. 文件或数据是否提供了清楚、合法、可核验的适用口径？
4. 是否存在与项目类型、采购标的、合同履约或监管数据不匹配的地方？
5. 是否有明确排除理由，例如法定要求、项目特殊需要、其他章节已补足或证据不足？

判断规则：
- 第 1、2 问为“是”，第 3 或第 4 问显示口径错误、不清或不匹配，且无排除理由时，进入命中候选。
- 证据不足但存在实质疑点时，输出待人工复核。
- 仅有关键词、没有实质后果时，不命中。

## 基础命中条件
以下 A、B、C 同时满足，即进入命中候选：
- A：存在与“质疑提交方式或材料要求过严”直接相关的条款、要求或交易数据线索。
- B：条款直接影响资格、评分、响应有效性、价格评审、合同履约、验收付款、供应商权利或政府采购政策适用。
- C：文件或数据未提供清楚、合法、可核验的解释，或存在明显错配、冲突、过度限制、不合理负担或监管异常。

## 命中条件
通常应同时满足以下条件，才认定命中本检查点：
- 候选证据与检查点标题直接相关。
- 风险后果不是抽象可能，而是会影响供应商参与、评审、政策适用、救济、履约验收、付款或监管判断。
- 文件内部或交易数据内部无法自行消除疑点。
- 不存在明确法定依据、项目特殊必要性或充分替代路径。

## 排除条件
任一满足，应排除或降级：
- 仅为背景说明、法定通用表述、已有明确补充条款、或不影响供应商参与和履约结果时，不命中。
- 条款已有明确合法依据，且与采购标的、项目特点和履约风险相适应。
- 其他章节已经补足适用对象、比例、期限、程序、标准、金额、范围或数据解释。
- 需要外部审批、预算批复、交易日志、投标文件或人工事实核验才能确认时，输出待人工复核。

## 判断结果分流
- 命中：基础命中条件 A、B、C 均满足，且无排除条件。
- 待人工复核：存在实质疑点，但需要外部政策、附件、审批、交易数据或人工事实核验。
- 不命中：只有关键词或背景表述，或者合法口径已经明确。

## 正例
- 条款将本检查点对象直接设置为资格、评分、价格优惠、响应有效性、付款或监管判断条件，但适用对象、比例、期限、标准、数据来源或责任边界不清。
- 同一事项在公告、正文、合同、附件或响应格式中口径不同，供应商无法据此稳定响应。
- 数据型检查点中，多个供应商之间出现同账户、同设备、同作者、规律性报价等异常一致，并有两个以上字段交叉印证。

## 反例
- 文件只是提示依法执行相关政策或程序，未形成额外限制或错误口径。
- 条款已经明确对象、范围、比例、期限、程序、标准和证明材料，且与项目事实相适应。
- 数据型检查点只有单一弱信号或缺少交易数据，不能直接定性。

## 审查步骤
1. 先检索对象词、后果词和组合信号。
2. 提取候选条款或交易数据，并读取上下文。
3. 判断条款属性和影响环节。
4. 执行专项判断尺子五问。
5. 检查排除条件和其他章节是否补足。
6. 按命中、待人工复核、不命中输出结论。

## 结论表达
- 命中时写明：原文或数据证据、错误口径、影响环节和法规政策边界。
- 待人工复核时写明：还需要补查的审批、附件、合同、平台日志、投标文件、交易数据或人工事实。
- 不命中时写明：为什么只是关键词出现，或为什么文件口径已经合法明确。

## 适用边界
质疑提交可以要求必要信息，但不得以过严形式要求实质限制救济；形式瑕疵不应当然剥夺质疑权。

## 备注
提交方式便利，材料要求必要合理；允许依法补正或多渠道提交，不以非实质形式排斥。
