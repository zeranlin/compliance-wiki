# NBD F1 金标校准链方案

## 1. 方案定位

本方案用于定义 NBD 质量评测中的标准答案校准过程。

NBD 工程审查结果需要通过召回率、精确率和 F1 值衡量，但原始标准答案通常存在三个问题：

```text
一个人工审查点可能包含多个 NBD 原子审查点。
工程额外命中的风险可能是真风险，但原始标准答案没有覆盖。
标准答案中的部分风险可能是弱口径、误标或证据不足。
```

因此，标准答案不能只停留在原始 V1，而应经过 Codex 裁判和 NBD 原子口径校准，形成可用于工程评测的最终金标。

## 2. 核心链路

标准答案校准分为四个版本：

```text
V1：原始标准答案
V2：原子化标准答案
V3：补充工程真风险后的标准答案
V4：裁判清洗后的最终评测标准答案
```

其中：

| 版本 | 含义 | 主要动作 |
|---|---|---|
| V1 | 人工原始标准答案 | 保留原始业务批注和人工判断 |
| V2 | 原子化标准答案 | 将复合审查点拆成 151 个 NBD 原子口径 |
| V3 | 补标标准答案 | 对工程命中但 V2 没有的项，由 Codex 裁判后补入真风险 |
| V4 | 最终评测标准答案 | 对工程未命中但 V3 中存在的项，由 Codex 裁判后删除或降级弱口径 |

## 3. V1 到 V2：原子化拆解

V1 是原始人工标准答案，可能一个审查点名称覆盖多个 NBD。

V1 到 V2 的目标是：

```text
把人工复合审查点拆成 151 个 NBD 可执行原子审查点。
```

处理规则：

1. 如果 V1 一个审查点实际包含多个 NBD 风险，应拆成多个原子项。
2. 如果 V1 审查点名称与 NBD 标题不完全一致，但风险本质一致，应映射到最合适的 NBD 原子审查点。
3. 如果 V1 中出现“新增”或不属于 151 NBD 的风险，应标记为新增，不参与 151 NBD 的 F1 计算。
4. 这一步只做拆解和映射，不补充工程额外命中，也不删除原始风险。

示例：

```text
V1：人员证书、业绩、认证要求不合理

V2 拆解：
NBD07-035 人员评审项是否与需求对应
NBD02-002 特定行业业绩限制
NBD03-002 体系认证证书不得要求特定认证范围
```

## 4. V2 到 V3：工程真风险补标

V2 评测时，工程可能输出了一些 V2 没有的风险。

这些风险不能直接都算误报，需要让 Codex 做裁判。

裁判规则：

| 工程额外输出 | Codex 裁判结论 | V3 处理 |
|---|---|---|
| 确是真风险，且属于 151 NBD 覆盖范围 | V2 漏标 | 补入 V3 |
| 确是真风险，但不属于 151 NBD 覆盖范围 | 覆盖范围外新增 | 标记新增，F1 忽略 |
| 不构成风险 | 工程误报 | 不补入 V3，保留为误报 |

这一步解决的是：

```text
标准答案漏标导致工程真风险被错误计算为误报。
```

## 5. V3 到 V4：弱口径和误标清洗

V3 中仍然可能存在工程没有命中的项目。

表面看这些都是工程漏报，但其中一部分可能是标准答案本身的问题，例如：

```text
证据不足。
风险口径过宽。
弱口径不能稳定成立。
行号偏移导致证据对应错误。
复合审查点中部分子项不能成立。
```

因此，工程未命中但 V3 中存在的项，需要再次由 Codex 裁判。

裁判规则：

| V3 中存在但工程未命中 | Codex 裁判结论 | V4 处理 |
|---|---|---|
| 确是真风险，且证据充分 | 工程漏报 | 保留在 V4 |
| 证据不足、弱口径或不能稳定成立 | 金标弱项 | 从 V4 删除或降级 |
| 真实存在但不属于 151 NBD | 覆盖范围外新增 | 标记新增，F1 忽略 |
| 行号偏移但风险成立 | 金标行号问题 | 修正行号后保留 |

这一步解决的是：

```text
标准答案过宽、弱口径或误标导致工程被错误计算为漏报。
```

## 6. 四类 F1 指标

完整评测可以形成四类指标：

| 指标 | 标准答案版本 | 用途 |
|---|---|---|
| F1-V1 | 原始标准答案 | 反映工程与原始人工口径的一致性 |
| F1-V2 | 原子化标准答案 | 反映工程与 151 NBD 原子口径的一致性 |
| F1-V3 | 补标后标准答案 | 反映工程真风险补入后的一致性 |
| F1-V4 | 最终裁判清洗金标 | 作为最可信的工程质量评测口径 |

汇报时可以保留 V1/V2/V3/V4 的完整链路。

工程文件命名可以继续使用 `V2.0.json`，但应在文档中注明：

```text
V2.0 金标内部实际经历：
原子化 -> 补标 -> 清洗
```

为了沟通清晰，建议方案汇报中使用：

```text
V1 -> V2 -> V3 -> V4
```

## 7. 边界规则

不允许：

- 不允许为了提高 F1 任意补充金标。
- 不允许为了提高 F1 任意删除漏报项。
- 不允许把工程额外输出直接视为真风险。
- 不允许把标准答案中的弱口径不经裁判直接保留。
- 不允许把 151 NBD 覆盖范围外的新增风险混入 F1。

必须做到：

- 每一次补入 V3 都要有 Codex 裁判理由。
- 每一次从 V4 删除、降级或忽略都要有 Codex 裁判理由。
- 每个金标项都要能对应行号、风险原文和 NBD 原子审查点。
- V4 是评测标准答案，不是业务审查输出。

## 8. 与 NBD 质量工程体系的关系

NBD 质量工程体系中：

```text
业务审查层负责产生工程输出。
质量评测层负责计算 V1/V2/V3/V4 指标。
建设治理层负责 Codex 裁判、补标、清洗和归因。
```

V1 到 V4 的金标校准链属于质量评测层和建设治理层，不进入日常业务审查链路。

最终目标：

```text
让 F1 不只是机械对比原始标准答案，而是衡量工程审查结果与真实、可执行、可复盘 NBD 原子金标之间的一致性。
```
