## 7. 干净边界

### 7.1 Python runtime 边界

Python 可以做：

1. 编译 Document IR。
2. 编译 NBD IR。
3. 按 NBD IR 解释召回配置。
4. 生成 CandidateWindow。
5. 生成 Prompt。
6. 调用模型。
7. 后处理通用 JSON 输出。
8. 评测召回率、精确率和 F1 值。

Python 不得做：

1. 不得写死具体样本名、案例号、标准答案行号。
2. 不得写死具体 NBD ID 分支。
3. 不得把某个 NBD 的业务命中条件写进 runtime。
4. 不得把评测标准答案变成运行规则。
5. 不得因为某个样本短期提分而破坏通用运行时。

### 7.2 NBD 边界

NBD 可以做：

1. 定义审查目标。
2. 定义定位与召回剖面。
3. 定义机器召回配置。
4. 定义候选召回规则和上下文读取规则。
5. 定义命中条件、排除条件、待人工复核边界。
6. 定义风险提示和修改建议。

NBD 不得做：

1. 不得写入案例一到八的具体行号。
2. 不得写入标准答案专用表达。
3. 不得只为某个样本的某个句子硬补命中。
4. 不得用老 BD 历史痕迹替代当前 NBD 的清晰表达。

### 7.3 评测层边界

评测脚本可以做：

1. 读取标准答案 JSON。
2. 忽略“新增”审查点。
3. 统计召回率、精确率和 F1 值。
4. 输出漏报、误报、归因诊断。
5. 做名称映射诊断。

评测脚本不得反向影响日常审查 runtime。
