# NBD 瘦身与 IR 拆分任务清单

## 1. 任务背景

当前 153 个 NBD 均已进入 `maintained`，并已完成 P4 单样本三轮对齐回归、P5 五品目真实文件扩展回归。

但 NBD 页面仍存在结构效率问题：

1. NBD markdown 同时承载运行知识、治理记录、历史调试备注和旧验证占位。
2. `prompt_runner` 当前仍主要使用 `compact_text`，会把较多非判断信息带入 prompt。
3. `定位与召回剖面` 与 `机器召回配置` 大量重复。
4. 141 个 maintained NBD 页面仍残留 `待 smoke / 待预检 / 待验证` 等历史占位，容易误导业务理解。

本任务目标不是重做 NBD，而是把 NBD 从“完整说明书”整理成“可编译源码”：

```text
NBD markdown 源码
  -> Recall IR：给 CLI 找候选窗口
  -> Prompt IR：给小模型执行 SOP
  -> Governance IR：给人和审计看，不进入 prompt
```

## 2. 升级原则

### 2.1 必须保持的边界

1. NBD markdown 仍是唯一知识源码。
2. CLI runtime 只解释 IR，不写具体 NBD 业务逻辑。
3. 召回知识进入 Recall IR，不直接塞给小模型。
4. 判断知识进入 Prompt IR，只保留小模型三分流所需内容。
5. 治理记录进入 Governance IR 或 audit 页，不进入日常审查 prompt。
6. 后处理只聚合和质检，不重新审查，不改写模型 verdict。

### 2.2 效率目标

1. 单 NBD prompt 中 NBD SOP 文本明显缩短。
2. 小模型看到的是结构化判断协议，而不是整页说明书。
3. 召回效果不退化，候选窗口仍可回溯到 NBD 源码。
4. P4、P5 回归结果不出现大面积退化。

## 3. 总体分层

| 层级 | 保留内容 | 编译目标 | 是否进入 prompt |
|---|---|---|---|
| frontmatter | id、title、status、scope、risk_level、version | 所有 IR metadata | 少量进入 |
| 审查定义 | 审查目标、适用范围、不适用范围 | Prompt IR | 进入 |
| 召回配置 | 主召回词、正式证据词、噪声词、冲突词、完整性要素、窗口扩展 | Recall IR | 不直接进入 |
| 候选窗口规则 | primary/support、模板降权、上下文读取 | Recall IR + Prompt IR 摘要 | 摘要进入 |
| SOP 判断 | 前置条件、命中条件、排除条件、待复核边界、结果分流 | Prompt IR | 进入 |
| 输出模板 | 风险提示、修改建议、审查依据 | Prompt IR + Report | 进入 |
| 治理记录 | 来源、验证记录、变更记录、旧 smoke 信息 | Governance IR / audit | 不进入 |
