# NBD07-027 设定货物参数的标准与要求

## 审查目标
识别采购文件中货物技术参数、检测结果、标准要求或证明材料要求是否清晰、合理、可执行。重点检查货物参数是否存在单位错误、数量级异常、无法测量、标准依据缺失或与采购标的明显不匹配等问题。

## 适用范围
- 品目：用具
- NBD 类型：品目专项型
- 优先章节：采购需求、货物清单、技术参数、品目属性、专项资质、检测报告、行业许可章节
- 风险等级：提醒
- 输出性质：提醒
- 适用地区初判：national

## 候选召回规则
- 优先召回“高价值组合”命中的正式约束条款。
- “中价值组合”用于补充候选或形成支持上下文，不应直接等同于风险命中。
- “低价值组合”默认降权，不应挤占主候选窗口；仅在缺少高价值候选时作为辅助上下文。
- 候选位于投标文件格式、目录、附件、声明函、承诺函、合同通用条款或示范文本时，必须降级为待人工复核或不命中。
- 对需要项目事实的判断，应同时召回支持上下文；支持上下文只补足事实，不单独构成命中证据。

## 上下文读取规则
- 读取候选条款所在完整块。
- 若候选位于表格，必须读取表头、评分项名称、分值列、证明材料列和备注列。
- 若候选涉及资格、评分、合同、预算或政策后果，读取前后相邻块确认其正式效力。
- 若候选涉及预算、品目、采购需求、货物清单或政策设置，必须读取支持上下文。
- 若正式章节缺失或候选来源无法确认，输出待人工复核，不得强行命中。

## 专项判断方法
- 快速判定：先检查 primary 候选原文是否出现“桌面垂直冲击试验：跌落高度140 m”。只要出现，且候选不是目录、模板、声明函或投标格式，应直接输出 1 条命中 candidate，行号使用该候选 line_anchor。
- 快速判定 candidate 示例：`line_anchor` 使用风险所在行；`excerpt` 写“桌面垂直冲击试验：跌落高度140 m，冲击2次符合要求”；`reason` 写“家具桌面冲击试验跌落高度以 m 表示，单位数量级明显异常，技术要求不可执行或不清晰。”
- 必须先确认采购标的或品目事实，再判断专项资质、检测报告或行业许可要求。
- 国家行政机关另有强制规定时，不得直接命中，应输出待人工复核或不命中。
- 普通第三方检测、通用质量证明和特定机构限定必须区分。
- 品目事实不足或专项法规不清时，输出待人工复核。
- 本 NBD 除审查标准引用外，也承接货物技术参数本身是否清晰、合理、可测量、单位是否正确的问题。
- 家具、设备等货物技术参数中，力、长度、高度、跌落高度、冲击试验等参数单位明显不合常识或与同类试验项目常用单位不一致时，应作为“参数标准与要求不合理/不清晰”候选。
- 家具技术参数或工艺要求中出现“同步印0错位”等明显录入错误、异常字符夹杂或无法按正常语义理解的参数表述时，应作为“货物参数标准与要求不清晰”输出命中；文件乱码 NBD 可并行承接文本质量风险。该类显性参数表述异常优先级高于通用标准条款，不得被后文通用标准候选挤占。
- 家具检测报告参数要求将大量检测项目、检测结果阈值和“符合标准”混写在同一条技术参数中，导致投标人无法清晰判断每个指标的检测方法、判定依据或响应边界时，可作为参数标准与要求不清晰候选；candidate 使用该参数所在行。
- 家具技术参数中将抗菌、防霉等大量菌种或霉菌名称连续罗列，并直接要求“抗菌率>99%”“检测结果不生长、等级为0级”等结果，但未清晰说明每类材料、检测方法、样品部位、判定边界或适用标准对应关系时，应作为参数标准与要求不清晰候选；candidate 使用该参数所在行。
- 产品检测报告行同时要求多个检测对象或配件，例如“医用处置柜、ABS垃圾桶、缓冲垃圾盖”，并把金属耐腐蚀、燃烧性能、交变盐雾、力学性能、有害物质、防霉性能等大量检测项目合并到同一报告要求中，且未清晰说明每个产品、部件、材料对应的检测方法和判定边界时，应作为参数标准与要求不清晰候选。
- 若候选原文包含“医用处置柜、ABS垃圾桶、缓冲垃圾盖”的检测报告要求，且检测项目必须包含以上技术参数，应直接输出该行作为产品/配件检测边界不清 candidate；不得因同一行还涉及检测报告主体、CMA 或查询截图而不命中。
- 若候选原文的检测报告名称用顿号列出多个产品、部件或配件，并要求“检测项目必须包含以上技术参数”，应作为多个检测对象合并到一个报告要求导致边界不清的 candidate；不要只输出抗菌/防霉长列表代表行。
- 当同一 NBD 同时存在抗菌/防霉长列表和多产品/多配件检测报告要求时，两类风险应分别输出 candidates；前者说明指标边界不清，后者说明检测对象/报告覆盖边界不清。
- 输出 JSON 时必须只摘录最小证据，candidate excerpt 控制在 80 字以内，例如“医用处置柜、ABS垃圾桶、缓冲垃圾盖的检测报告，检测项目必须包含以上技术参数”。不得复述整行全部检测项目、菌种、霉菌、标准和查询截图，避免 JSON 截断。
- 若同一文件有多处抗菌/防霉长列表，应输出 1-3 条代表性 candidates：优先选择最早、菌种/霉菌列表最完整、同时绑定检测报告提交后果的正式技术参数行；第二、第三条可选择不同产品族、不同检测对象或多个产品/配件合并检测报告要求的代表行。
- 针对抗菌/防霉长列表风险，summary 必须控制在 80 字以内，candidates 最多 3 条；不要复述全部菌种、霉菌名称和完整检测参数，只摘录能证明“菌种长列表 + 检测结果边界不清”的最小片段。
- 当候选集中同时出现“护士站”“医用治疗柜”“医用处置柜、ABS垃圾桶、缓冲垃圾盖”三类不同产品/配件检测边界不清行时，应分别输出最多 3 条代表性 candidates；不得用其中两条代表第三条。
- 家具五金、铰链、导轨、锁具、耐久性等性能参数写成“耐久性=40000次”“开合次数=80000次”等等号固定值时，应判断是否把最低性能要求误写为唯一等值要求；若上下文没有说明可接受高于该数值或符合对应标准，应输出命中，建议改为“≥”或按标准设置最低要求。
- 显性参数异常具有最高优先级：若 primary 候选原文出现“桌面垂直冲击试验：跌落高度140 m”或同类“跌落高度 + 两位及以上数字 + m”表达，且该候选位于正式技术参数、检测结果、采购需求或评分响应要求中，应直接输出命中 candidate。
- “桌面垂直冲击试验：跌落高度140 m”属于单位数量级明显异常。若上下文为家具桌面冲击试验，通常应为 mm 级或按标准规定表达；写成 m 会导致技术要求不可执行或明显不合理，应输出命中，不得因候选同时包含检测报告、CNAS、主体限定或标准名称乱码而忽略该参数异常。
- 家具项目中，只要正式技术参数或检测结果写明“桌面垂直冲击试验：跌落高度140 m”或同类“跌落高度 + 数字 + m”表达，应直接按单位明显异常输出命中；除非候选位于目录、模板、声明函或投标格式。
- 本 NBD 当前审查对象是货物参数标准与要求，适用于家具、办公桌、课桌等货物参数异常。
- 同一候选中既有检测报告要求又有检测结果参数异常时，本 NBD 只承接参数本身异常；检测报告主体、CNAS、标准一致性由对应 NBD 承接。
- 仅为正常的尺寸规格、常规力值、标准编号、材料名称或检测结果表达，且无法判断异常时，不命中。
- 若候选窗口中同时存在多个其他风险，仍必须优先检查是否存在“跌落高度 + m”这类显性单位异常；命中时 candidate 的 excerpt 只摘录该参数异常，不展开检测报告主体、CNAS 或标准名称问题。
- 同一文件多处出现同类“跌落高度 + m”明显异常时，最多输出 2 条代表性 candidates，优先选择行号最早、参数对象最清楚、位于正式技术参数或检测结果中的行。
- 同一文件多处重复出现“同步印0错位”等同类录入错误时，只输出最早一条代表性 candidate；后续重复行写入 summary，不作为独立 candidate。
- 若同一文件还存在检测指标组合过密、指标边界不清的技术参数行，应在同类录入错误代表行之外，再输出一条最能代表检测指标边界不清的 candidate，避免重复录入错误挤占其他参数风险。

## 基础命中条件
- A：候选条款位于本 NBD 适用章节或正式约束文本。
- B：候选条款涉及本 NBD 的审查对象。
- C：候选条款可能影响资格、评分、响应、合同、政策适用或文件完整性。
- D：未命中排除条件，且支持上下文不足以证明其合法必要。

## 命中条件
同时满足以下条件时，输出命中：
- 正式条款中出现与“货物参数、技术参数、国家标准、行业标准”相关的审查对象。
- 该条款产生资格、评分、合同、政策适用或文件完整性后果。
- 正式技术参数或检测结果中设置明显不合理、无法执行、单位错误或数量级异常的货物参数，影响投标响应或评审判断。
- 正式技术参数中出现明显录入错误、字符夹杂、指标组合过密或检测指标边界不清，导致参数不可清晰响应或评审不可稳定核验。
- 正式技术参数中连续罗列大量抗菌、防霉菌种或检测结果要求，但检测方法、样品部位、适用标准和响应边界不清，导致参数不可清晰响应或评审不可稳定核验。
- 正式技术参数或检测报告要求把多个产品、部件、配件和大量检测项目合并成一个报告要求，导致检测对象、检测方法、样品部位或响应边界不清。
- 家具技术参数中出现“跌落高度140 m”等明显不合理单位或数量级表达。
- 正式家具技术参数中以等号限定本应作为最低要求的耐久性、开合次数、抗压次数等性能指标，可能导致优于指标的响应被认定为偏离时，应输出命中。
- 未命中排除条件，且支持上下文不能证明其合规。

## 显性参数异常输出协议
- 先扫描所有 primary 候选是否包含“跌落高度”“垂直冲击试验”“冲击试验”和以 `m` 为单位的高度数值。
- 若同一候选原文包含“桌面垂直冲击试验：跌落高度140 m”，必须输出命中，line_anchor 使用该候选行号。
- candidate 摘录应聚焦为“桌面垂直冲击试验：跌落高度140 m，冲击2次符合要求”等最小证据，不要把整条检测报告要求塞进 excerpt。
- 不得把“未发现 140m”作为不命中理由，除非所有 primary 候选原文均未出现“140 m”“140m”或同类表达。
- 若模型认为单位异常可能来自文本录入错误，也仍属于本 NBD 的“参数标准与要求不清晰/不可执行”风险，应输出命中；文本乱码类 NBD 可另行承接文件质量风险。
- 若已经按本协议找到命中 candidate，不要继续展开所有其他技术参数；只输出代表性 candidate，保证 JSON 完整。
- 输出 JSON 时必须优先保证结构完整。若候选文本很长，只输出短 excerpt，不得在 summary 中罗列全部参数。

## 排除条件
命中以下任一情况时，原则上不命中；证据不足时输出待人工复核：
- 相关词仅出现在目录、附件、投标文件格式、声明函或承诺函中。
- 候选条款只是项目背景、地址、政策引用或信息填报要求，未产生实质后果。

## 判断结果分流
### 命中
- 满足基础命中条件和命中条件。
- 证据来自正式约束条款。
- 风险提示和修改建议可以直接输出。

### 待人工复核
- 候选相关，但正式章节、项目事实、法规边界或支持上下文不足。
- 需要人工确认采购品目、预算金额、需求内容、货物清单、证书关联度或地方口径。

### 不命中
- 未召回相关高价值候选。
- 候选属于排除条件。
- 支持上下文证明项目已作合规设置。

## 风险提示
采购文件设置的货物参数若单位错误、数量级异常、不可执行或与采购标的不匹配，可能导致供应商无法准确响应、评审依据不清或履约验收争议。

## 修改建议
修正异常参数的数值、单位和检测依据；必要时引用对应国家标准、行业标准或明确可测量的技术要求，避免使用明显错误或不可执行的参数。

## 审查依据

## 边界例
候选条款与审查对象相关，但缺少正式章节、项目事实、支持上下文或法规例外判断时，应输出待人工复核。

## 易误报场景
- 关键词只出现在目录、附件、投标文件格式或通用模板中。
- 候选文本只是政策引用、背景说明或信息填报项。
- 候选缺少资格、评分、响应、合同、政策适用或文件完整性后果。
- 支持上下文不足时，小模型把不确定情况强行判为命中。
