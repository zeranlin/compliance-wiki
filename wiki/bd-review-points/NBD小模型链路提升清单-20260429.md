# NBD 小模型链路提升清单 20260429

## 目标

实际业务链路只由小模型 CLI 执行审查。Codex-NBD 与 Codex 自由审查只用于辅助 NBD 和 CLI 调优，最终必须回到 153 个 NBD 与日常 CLI。

本轮提升目标：
- 小模型看得到关键证据。
- 小模型看得懂 NBD SOP。
- 小模型能区分正式条款和模板噪声。
- 小模型能用反证降噪。
- 小模型能在证据充分时把待复核升级为命中。
- 业务报告只展示业务人员需要处理的问题。

## 约束

- 153 个 NBD 总量不变。
- 不新增 NBD 编号。
- Codex 自由审查发现的新风险，归入最合适的现有 NBD。
- NBD 扩展只修改 SOP、关键词、命中条件、排除条件、反证召回和边界说明。
- CLI 只做候选窗口、反证窗口、结构化窗口和报告聚合，不写入第二套业务规则。

## 第一组：配置型/缺失型 NBD 降噪

| NBD | 问题 | 升级重点 |
|---|---|---|
| NBD01-017 | 服务地点 | 增加“履约地点下一行”“项目概况地址/楼宇位置”为反证；只看到标题不得判风险。 |
| NBD03-005 | 服务期限 | 正式商务要求中出现服务期限即不命中；排除评分项里的“服务期满”噪声。 |
| NBD04-003 | 所属行业 | 正式服务需求明细优先；声明函模板只能作线索，不能作缺失证据。 |
| NBD06-009 | 付款方式 | 已写付款期限、付款方式、结算日或凭票办理付款手续则不命中；与 NBD06-007 区分。 |
| NBD06-014 | 履约保证金退还方式 | 先判断是否收取履约担保；若明确“不需要”，直接不适用。 |
| NBD01-029 | 实质性条款 | 正式实质性条款表、星号条款、响应表均为反证；通用否决说明不能单独判缺失。 |
| NBD02-032 | 应急预案需求对应 | 必须同时召回评分项和采购需求；需求中已有应急处置则不命中。 |
| NBD01-033 | 错别字检查 | 排除“笔误处理规则”“澄清补正规则”等非错别字风险。 |

## 第二组：待复核应升级的风险型 NBD

| NBD | 问题 | 升级重点 |
|---|---|---|
| NBD02-004 | 特定行政区划奖项 | 覆盖“副省级及以上政府部门/人民政府颁发”等行政层级限定。 |
| NBD02-005 | 特定行业奖项 | 覆盖“物业管理安全类荣誉”“消防工作先进单位类荣誉”等行业荣誉。 |
| NBD02-014 | 证照/资质隐性限制 | 扩展为证照、资质、人员证书堆叠造成隐性限制。 |
| NBD03-001 | 证书设置合理性 | 增加证书、认证、荣誉集中评分的综合合理性判断。 |
| NBD06-007 | 发票后 10 个工作日付款 | 已写付款方式但未写收到发票后 10 个工作日，按风险命中或高优复核。 |
| NBD06-006 | 分包内容/金额/比例 | 明确允许分包但未写金额或比例，可升级为命中。 |
| NBD07-018 | 高处作业证 | 覆盖外墙维修、楼宇外墙清洗、小区外墙清洗等高处作业线索。 |
| NBD07-035 | 人员评审与需求对应 | 扩展人员学历、职称、证书、经验、年龄、性别、身高、退伍军人、公检法系统经验等与需求不对应问题。 |

## 第三组：153 内覆盖扩展

| 新风险 | 归入 NBD | 扩展重点 |
|---|---|---|
| 人员性别、年龄、身高限制 | NBD07-035 | 作为人员配置或人员评分与需求对应性问题。 |
| 退伍军人、公检法系统经验限制 | NBD07-035 | 判断是否超过岗位必要性。 |
| 人员证书、职称、学历组合过高 | NBD07-035 / NBD02-014 | 人员侧归 NBD07-035，证照堆叠归 NBD02-014。 |
| 弱相关软件系统评分 | NBD07-028 | 扩展为与采购需求不相符的资质、系统、证书、证明材料。 |
| 软件著作权/知识产权证明评分 | NBD02-034 | 不只看“自有”，也看知识产权证明作为服务能力评分是否合理。 |
| 参与行业标准编写评分 | NBD07-028 / NBD02-005 | 弱相关证明归 NBD07-028，行业荣誉身份归 NBD02-005。 |
| 证书、认证、荣誉集中高分 | NBD03-001 | 增加“同一评分表内证书/荣誉密度过高”判断。 |

## 第四组：CLI 反证召回升级

| 场景 | CLI 召回升级 |
|---|---|
| 标题型窗口 | 命中“履约地点”“服务期限”“付款方式”等标题时，自动带后 1-3 个块。 |
| 配置型 NBD | 同时召回疑似缺失窗口和正式配置反证窗口。 |
| 模板噪声 | 投标文件格式、声明函、通用条款降权。 |
| 正式条款 | 用户需求书、商务要求、实质性条款、服务需求明细升权。 |
| 付款类 | 同时召回付款方式、付款期限、发票、结算日。 |
| 分包类 | 同时召回正式分包条款、分包意向协议模板、中小企业价格扣除段落。 |
| 应急预案类 | 同时召回评分项和需求侧应急处置内容。 |
| 人员类 | 同时召回评分人员条件和需求侧人员最低配置。 |

## 第五组：评分表结构化升级

| 能力 | 作用 |
|---|---|
| 整表窗口 | 保留价格、技术、商务、诚信总结构。 |
| 分项窗口 | 每个评分项独立成块。 |
| 评分内容 + 评分依据绑定 | 防止只看到内容看不到证明材料。 |
| 总分校验 | 支持 NBD02-030 降噪。 |
| 子项加总校验 | 判断技术/商务分值是否完整。 |
| 证书/荣誉/人员/业绩密度统计 | 支持证书堆叠和隐性限制判断。 |
| 需求-评分对照 | 支持 NBD07-035、NBD02-032、NBD07-028。 |

## 第六组：业务报告升级

| 场景 | 报告处理 |
|---|---|
| 同一原文触发多个 NBD | 合并成一个业务问题。 |
| 命中 + 待复核同源 | 放到同一个问题下，不重复展示。 |
| 已被反证排除 | 不进入业务报告正文，可进调试报告。 |
| 模板噪声 | 不进入业务报告正文。 |
| 证书/荣誉/人员条件集中问题 | 聚合展示，不拆成多个碎问题。 |
| 待复核 | 必须写清“业务人员需要确认什么”。 |

## 回归验证清单

必须继续命中：
- 本地经营（服务）网点进入评分。
- 党政机关或事业单位办公楼类物业业绩。
- 方案类优良中差主观分档。
- 安全生产标准化证书评分。

应从待复核降噪：
- 服务地点缺失。
- 服务期限缺失。
- 所属行业缺失。
- 付款方式缺失。
- 履约保证金退还方式缺失。
- 实质性条款缺失。
- 错别字风险。
- 应急预案需求缺失。
- 技术/商务评审项不完整。

应从待复核升级或更明确：
- 特定行业/行政层级荣誉评分。
- 多个认证证书叠加评分。
- 付款时限未明确 10 个工作日。
- 分包内容有但金额/比例缺失。
- 外墙清洗/维修高处作业证。
- 人员评分与需求不对应。

应由 153 内扩展覆盖：
- 人员性别、年龄、身高。
- 退伍军人、公检法系统经验。
- 弱相关软件系统评分。
- 软件著作权/知识产权证明评分。
- 参与行业标准编写评分。
