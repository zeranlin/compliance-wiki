NBD 可迭代质量工程体系架构图

本图用于汇报 NBD 如何从法规与业务经验沉淀为可运行审查知识，并通过业务审查层、质量评测层、建设治理层形成可量化、可复盘、可持续提升的闭环。

业务审查层

  • • 负责日常待审文件的 151 个 NBD 全量审查
  • • 输出业务人员可读的风险问题、证据位置和复核建议
  • • 不读取标准答案，不把评测逻辑带入业务报告

质量评测层

  • • 用 V1/V2 标准答案计算召回率、精确率和 F1 值
  • • V2 以 151 个 NBD 原子审查点为主口径
  • • 负责漏报误报清单、工程 JSON 和样本汇总报告

建设治理层

  • • Codex 作为裁判和归因者，不进入日常审查链路
  • • 判断改 NBD、改通用运行时，还是校准 V2 金标
  • • 守住边界，防止 CLI 长成第二套审查规则库
