NBD F1 金标校准链架构图

本图用于说明 F1 评测中的标准答案如何从原始 V1 逐步变成最终 V4：先原子化，再由 Codex 裁判补入工程真风险，最后清洗弱口径和误标，形成更可信的工程质量评测金标。

为什么需要 V1 到 V4

  • • 原始标准答案常有复合审查点、漏标和弱口径
  • • 直接用 V1 计算 F1，会把工程真风险算成误报
  • • 也可能把金标弱项错误算成工程漏报

Codex 的位置

  • • Codex 不进入日常业务审查链路
  • • 只在质量评测和建设治理层做裁判
  • • 负责判断补入 V3、保留漏报、删除弱口径或标记新增

最终口径

  • • 汇报时建议讲清 V1 -> V2 -> V3 -> V4
  • • 工程文件可继续用 V2.0，但需说明内部阶段
  • • V4 是最可信的工程质量评测金标
