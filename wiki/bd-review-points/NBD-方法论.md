## 定位
NBD 是面向小模型执行的标准 BD 检查点说明书，用于从政府采购文件中召回候选条款、判断风险、输出依据、风险提示和修改建议。

NBD 以业务方新版审查材料为孵化来源，经法规映射、风险归因和 SOP 标准化后形成。NBD 与旧版 `checkpoints` 完全隔离；旧版材料可作为建设过程中的参考素材，但不在 NBD 页面中体现。

## 建设目标
- 让小模型能快速找到候选条款。
- 让小模型能按条件判断命中、待复核或不命中。
- 让输出结果包含风险点、证据、风险提示、修改建议和审查依据。
- 支持跨项目、跨品目和跨地区复用。
- 支持持续调试和迭代，不把模板做成僵硬表单。

## 拆分原则
一个 NBD 应对应一个稳定、可执行、可判断的审查问题。

基本公式：

```text
一个 NBD = 一个风险对象 + 一个适用位置 + 一套定位与召回剖面 + 一组命中条件 + 一组排除条件 + 一个输出动作
```

需要拆分的情形：
- 风险对象不同。
- 适用章节不同，例如资格条件、评分因素、采购需求、合同条款、公告信息。
- 法规依据或地区口径不同。
- 命中条件不同。
- 排除条件不同。
- 风险提示或修改建议不同。

可以合并的情形：
- 只是同义表达不同。
- 风险本质相同。
- 只是案例不同。
- 只是品目表述略有差异，判断逻辑和输出动作一致。

## 页面结构
NBD 页面分为稳定执行区和弹性迭代区。

稳定执行区供脚本和小模型读取，应尽量保持一致：

```text
审查目标
适用范围
定位与召回剖面
候选召回规则
上下文读取规则
基础命中条件
命中条件
排除条件
判断结果分流
风险提示
修改建议
审查依据
```

弹性迭代区可按需要增删：

```text
示例
反例
易误报场景
品目专项说明
地区适用说明
调试备注
小模型可执行性自检
```

## 定位与召回剖面
NBD 页面必须使用“定位与召回剖面”，不再使用“定位关键词”作为标准章节名。

“定位关键词”容易把建设方向带回固定词表；新版 NBD 应直接表达为面向未知采购文件的召回剖面。

未来采购文件的章节名称、表格结构、措辞和顺序都可能变化，因此 NBD 不得只依赖某几个范本文档中的固定表述。定位应由五层组成：

```text
词簇层 + 模式层 + 章节角色层 + 组合评分层 + 支持上下文层
```

### 词簇层
词簇层用于覆盖未知文件中的同义词、近义词、业务别名和地区叫法。

每组词应区分强词、中词、弱词和噪声词：

- 强词：单独出现就具有较强指向性。
- 中词：需要与章节、行为或后果组合。
- 弱词：只能辅助召回，不应单独形成高价值候选。
- 噪声词：出现后应降权或转入排除判断。

例如“投标保证金”：

```text
强词：投标保证金、响应保证金、投标保函
中词：保证金、担保、保函
弱词：金额、比例、提交、退还
噪声词：履约保证金、质量保证金、农民工工资保证金
```

### 模式层
模式层用于抽取文本形态，不直接作风险结论。

常见模式包括：

```text
金额：\d+(\.\d+)?\s*万元
比例：\d+(\.\d+)?\s*%
年限：近[一二三四五六七八九十\d]+年
分值：得\d+(\.\d+)?分、不得分、满分\d+(\.\d+)?分
标准号：GB/T、GB、ISO、ASTM、BIFMA 等标准代号
门槛：不低于、不少于、高于、低于、以上、以下
```

模式层只能提示“这里有金额、比例、年限、分值或标准号”，风险判断仍需结合章节角色、对象和后果。

### 章节角色层
章节角色层用于识别文本在采购文件中的功能，而不是硬匹配标题。

例如“资格条件”可能写成：

```text
申请人的资格要求
投标人资格要求
供应商资格
响应人资格
合格投标人
参与条件
准入要求
资格审查
```

“评分标准”可能写成：

```text
评审因素
评审标准
评分细则
综合评分表
商务部分
技术部分
价格部分
评分准则
```

章节角色应结合标题、邻近结构、表头和后果词综合判断。例如同时出现“项目概况、预算金额、最高限价、采购需求、合同履行期限、本项目是否接受联合体”，即使标题不叫“招标公告”，也应识别为公告或项目基本情况区域。

### 组合评分层
候选召回不得按“命中一个词就召回”处理，应按组合判断候选价值。

高价值候选：

```text
章节角色 + 对象词簇 + 行为词簇
对象词簇 + 行为词簇 + 后果词簇
评分项 + 对象词簇 + 得分结构
对象词簇 + 数值模式 + 后果词簇
```

中价值候选：

```text
对象词簇 + 行为词簇
对象词簇 + 数值模式
章节角色 + 对象词簇
```

低价值候选：

```text
只有对象词
只有章节词
只出现在目录、格式、附件、模板、承诺函或示范文本中
```

低价值候选不应直接挤占主候选窗口，可作为辅助上下文或备用窗口。

### 支持上下文层
部分 NBD 的风险证据和判断事实不在同一窗口内，必须召回支持上下文。

支持上下文用于补足项目事实，例如：

- 预算金额。
- 采购品目。
- 是否专门面向中小企业。
- 招标公告正式段落。
- 采购需求、技术参数、货物清单。
- 评分项对应的需求依据。

支持上下文不单独构成风险证据。最终命中仍必须回到正式约束条款、命中条件和排除条件。

## 小模型执行链
每个 NBD 应能支撑小模型按以下流程执行：

1. 锁定章节：先判断应优先读取资格条件、评分表、采购需求、合同、公告或其他章节。
2. 召回候选：根据定位与召回剖面圈定高价值候选条款，并补充必要支持上下文。
3. 读取上下文：按 NBD 要求读取候选条款所在块及必要前后文。
4. 识别条款属性：判断候选条款属于资格条件、评分因素、采购需求、证明材料、履约要求、合同条款、公告信息、模板残留或其他。
5. 核对命中条件：逐项判断基础命中条件和核心命中条件。
6. 核对排除条件：排除格式填报、模板残留、合法必要条件、非正式文本等误报。
7. 输出结论：给出命中、待人工复核或不命中，并说明证据、风险提示、修改建议和依据。

## 合理性判断型 NBD
合理性判断型 NBD 不能只靠关键词命中，因为风险通常来自“对象与项目需求之间缺乏必要关联”，而不是对象本身违法。

此类 NBD 必须采用“逐对象关联度判断”：

1. 先拆对象：同一评分项或资格条件内有多个证书、资质、荣誉、认证、业绩或设备要求时，必须逐个判断。
2. 判对象性质：区分行政许可、强制认证、执业资格、岗位能力、产品或服务认证、管理体系认证、荣誉称号、企业身份、协会评价等。
3. 找需求锚点：回到采购标的、服务内容、技术参数、岗位人员、作业场景、货物清单或依法准入要求。
4. 判关联强度：直接对应、表面相关但必要性不明、明显无关三档。
5. 判后果强度：资格准入、响应无效、高分值评分、普通低分值评分应分开。
6. 再分流结论：明显无关且有准入或高分值后果才命中；表面相关但必要性或分值合理性不明时待人工复核；依法必要或直接对应需求时不命中。

小模型不得把同一条款中的所有对象合并成一个结论。例如证书评分项中同时出现环保认证、产品认证、社会责任体系认证时，应逐项判断哪些可能合理、哪些可能无关。

## 结论分流
小模型输出外层结论保持三类：

```text
命中 / 待人工复核 / 不命中
```

NBD 页面应同时保留风险等级和输出性质：

```yaml
risk_level: 高 | 中 | 低 | 提醒
finding_type: 风险 | 提醒 | 配置缺失 | 口径错误 | 编制建议
```

## 地区适用边界
NBD 应区分全国通用规则、地方口径规则和混合规则。

```yaml
standard_scope: national | regional | mixed
source_region: 深圳
applicable_regions:
  - 全国
regional_notes:
```

判断原则：
- 全国法规、部门规章或全国性规范性文件能支撑的，标记为 `national`。
- 仅地方政策或地方交易规则支撑的，标记为 `regional`。
- 全国原则加地方细化口径的，标记为 `mixed`。

地方口径不得直接包装为全国通用规则。

## 状态定义
- `draft`：已从业务材料孵化，尚未完成充分法规映射和小模型验证。
- `testing`：已具备基本 SOP，可进入小模型验证和样本调试。
- `maintained`：已完成法规依据、命中条件、排除条件和小模型可执行性验证。

## 小模型可读性要求
- 每段尽量短。
- 条件使用列表表达。
- 命中条件应可逐项打勾。
- 排除条件必须明确。
- 少用无法执行的抽象表述。
- 不确定情况应导向待人工复核。
- 法规依据放在判断规则之后，不用长篇法规解释压过执行链。
- 示例和反例应短而典型。

## 小模型可执行性自检
每个 NBD 进入 `testing` 前，应至少完成以下自检：

- 只读“定位与召回剖面”中的词簇和模式，能否覆盖未知文档的同义表述。
- 是否定义了章节角色，而不是只硬匹配范文标题。
- 是否定义了高价值、中价值、低价值候选组合。
- 是否定义了降权或排除词，避免格式、模板、目录和附件噪声。
- 是否定义了必要支持上下文。
- 只读“候选召回规则”能否圈定合理候选。
- 只读“命中条件”能否判断风险。
- 是否有排除条件减少误报。
- 是否能输出明确风险提示和修改建议。
- 是否能说明审查依据。
- 是否明确地区适用边界。

## 建设节奏
NBD 不追求一次性完美，应通过样例试写、小模型验证、误报漏报修正和批量生成逐步完成。

推荐流程：

1. 先确定目录结构和页面模板。
2. 选择代表性审查点试写。
3. 用小模型验证候选召回和判断结果。
4. 根据误报、漏报和输出质量调整模板。
5. 批量生成剩余 NBD。
6. 做法规映射、地区边界和小模型可执行性验收。
